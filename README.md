# 制作背景
学習のアウトプットの目的で作成したアプリですが、Nuxt.jsとLaravelの基礎力定着以外に意識した点が２点ほどあります。<br>
①ログイン後、メモの追加、編集、削除が少ない操作でできること。<br>
②初めて使う方、パソコンが苦手な方でも直感的に簡単に操作ができるような設計。<br>
メモアプリなのでメモを取ること自体のハードルが下がるような設計を意識しました。<br>

前の職場ではパソコンに苦手意識を持っている方も多かったので、そういった方でも感覚的に操作できる点を１番意識をして<br>
画面を作成しました。

# 工夫した点
## ●メモの追加や削除時に即時に画面に反映
メモの追加や削除の操作時にリロードしなくても画面に反映できるよう、<br>
findIndex（）→splice（）、unshift（）などを使用しました<br><br>

## ●フラッシュメッセージを連続して表示（リロードしなくても表示）できるように
propsやemitの使用ではログイン（メッセージ表示）→メモ追加（メッセージが表示されない）といった問題がありました。<br>
そのためpiniaを使用して状態を管理し、連続してメッセージを出せるようにしました。<br><br>

# 苦労した点
## ●認証機能で以前使用できていたモジュールが使用できない
今回はNuxt3で開発をしたため、Nuxt２で使用ができていた認証用のモジュールが使用できず、<br>
Laravelのsanctumとの相性も考慮しnuxtSanctumAuthというモジュールを使用しました。<br>
英語の記事が数個しかなく情報が少なかったですが、ログを出しながら開発を進めていき、何とか実装することができました。<br><br>

## ●本番環境で認証ができない
今回Nuxt.jsをvercelで、LaravelをAWSでデプロイしたのですが、開発環境ではログインできていたのに本番環境ではログインできない問題が発生しました。
コンソールのエラーメッセージから、開発環境のモジュールの設定の修正と開発環境の.envファイルの修正を行いましたがそれでも解決ができず、原因の把握に
時間がかかりましたが、AWSで使用したドメインのサブドメインを作成→vercelにサブドメインを紐づけることで解決できました。
※ただいま動作確認中です


## ●Nuxt3の情報が少ない
Nuxt3が私の想定以上に情報がまだ少なく、基本的な実装でも苦労しました。<br>
次回からもっとスムーズに開発ができるよう、認証機能やパラメーター、バリデーションについてなど<br>
実装した機能についてNotionというサービスを使用して参考にした記事、コードをまとめながら作業を進めていきました。<br><br>
